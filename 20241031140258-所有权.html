所有权

基础

堆 Stack & 栈 Heap

堆 杂乱无序

  • 大小未知,可能变化,速度慢
  • 堆内存是从低位地址向上增长
  • 堆内存大小只受物理内存限制

栈 像瓶子 后进先出

  • 所有数据都必须占用固定大小的内存,方便分割内存
  • 调用函数时,实参被压入栈中
  • 栈内存从高位地址向下增长
  • Rust 中 main 线程的栈大小是 8mb,普通是 2mb

变量再堆栈的分配

变量再堆栈的分配位置主要取决于其类型和大小

  • 基本数据类型都是存在栈上,还有较小的复合类型(如元组.数组.切片等)
  • 动态大小的数据类型存放再堆上,如 String,Vec,HashMap,智能指针等

原则

  1. rust 中每个值都只能被一个变量所拥有
    • 否则会造成二次释放
    • 所以才有变量遮蔽
  2. 当所有者离开作用域时,这个值被丢弃
    • 自动调用 drop 函数清理堆内存,解决

问题

悬垂引用

  • 返回值的类型是地址类型的时候就要注意了,可能会引起悬垂引用
  • 引用的对象比它的引用者存在的时间更短
    即返回地址时,地址上的值被 rust 的 GC 销毁了或者更改了,反正不再是原来的值了.
  • 第一个例子
    int* foo() {
    int a;          // 变量a的作用域开始
    a = 100;
    char *c = "xyz";   // 变量c的作用域开始
    return &a;
}                   // 变量a和c的作用域结束

    返回了 a 地址,但 a 地址所在的值已经被销毁了.此时 a 已经无法被外部调用

  • 第二个例子
    fn main() {
    let reference_to_nothing = dangle();
}

fn dangle() -> &String {
    let s = String::from("hello");
    &s
}

    返回 s 的地址,但 s 已经销毁了

二次释放

let s1 = String::from("hello");
let s2 = s1;

当 s1 和 s2 离开作用域，它们都会尝试释放相同的内存。这是一个叫做 二次释放（double free）的错误

解决

  • 单一所有权 ： Rust 严格保证每个值在任意时刻只有一个所有者，从而避免了二次释放问题。
  • 所有权转移 ： 当一个变量被赋值给另一个变量时，所有权会从原变量转移到新变量，原变量将不再有效。

转移所有权(移动)

  • 标量类型都是通过自动拷贝来复制的, 复杂类型才会发生转移

      let x = 5;
  let y = x;

  • 复杂类型会导致二次释放

      let s1 = String::from("hello");
  let s2 = s1;
  println!("{}, world!", s1);
    • s1 被 移动 到了 s2

克隆

浅拷贝

fn main() {
    let x: &str = "hello, world";
    let y = x;
    println!("{},{}",x,y);
}

因为 x 只是获得了值的地址,所以进行了浅克隆,x 和 y 都获得了值的引用

let s1 = String::from("hello");

println!("s1 = {}, s2 = {}", s1, s2);

函数传值与返回

函数传值

  • 函数参数的三种传入方式:转移所有权(移动)、可变引用、不可变引用
  • 复杂类型
let s = String::from("hello");  // s 进入作用域
takes_ownership(s); // s 不再有效
  • 标量类型
let x = 5;                      // x 进入作用域
makes_copy(x); // x 后续还可以被使用

返回值

fn main() {
    let s1 = gives_ownership();         // gives_ownership 将返回值
                                        // 移给 s1

    let s2 = String::from("hello");     // s2 进入作用域

    let s3 = takes_and_gives_back(s2);  // s2 被移动到
                                        // takes_and_gives_back 中,
                                        // 它也将返回值移给 s3
} // 这里, s3 移出作用域并被丢弃。s2 也移出作用域，但已被移走，
  // 所以什么也不会发生。s1 移出作用域并被丢弃

fn gives_ownership() -> String {             // gives_ownership 将返回值移动给
                                             // 调用它的函数

    let some_string = String::from("hello"); // some_string 进入作用域.

    some_string                              // 返回 some_string 并移出给调用的函数
}

// takes_and_gives_back 将传入字符串并返回该值
fn takes_and_gives_back(a_string: String) -> String { // a_string 进入作用域

    a_string  // 返回 a_string 并移出给调用的函数
}

借用:获取变量的引用 = &var

& 引用 与 * 解引用

fn main() {
    let x = 5;
    let y = &x;

    assert_eq!(5, x);

再借用

&* 组合使用解引用与引用

  • 破坏借用规则,使用可变引用与不可变引用同时存在
struct Point{}
impl Point{ fn move_to(&mut self, x:i32, y: i32){...} }
fn main(){
    let mut p = Point { x: 0, y: 0 };
    let r = &mut p;
    // reborrow! 此时对`r`的再借用不会导致跟上面的借用冲突
    let rr: &Point = &*r;
    //再借用`rr`最后一次使用发生在这里，在它的生命周期中，我们并没有使用原来的借用`r`，因此不会报错
    println!("{:?}", rr);

    // 再借用结束后，才去使用原来的借用`r`
    r.move_to(10, 10);
    println!("{:?}", r);
}

不能在再借用的生命周期内使用可变引用

  • 强制类型转换
let string = String::from("hello");
let string_slice: &str = &*string;  // Coerce String to str
  • 可以将智能指针转换为引用
let boxed = Box::new(5);   // Box<i32>
let reference = &*boxed;   // &i32 (首先解引用 Box 得到 T，然后取它的引用)
  • 可以消除嵌套引用
let x = 5;
let y = &x;    // &i32
let z = &&y;   // &&i32
let w = &*z;   // &i32 (首先解引用一次，然后取引用)
  • 特征
trait MyTrait {}
let boxed: Box<dyn MyTrait> = // ...
let reference: &dyn MyTrait = &*boxed;

不可变引用

不变性使复杂的功能更容易实现,避免数据直接突变可以让你保持以前版本的数据完好无损，并在以后重用它们

:ID: cd590188-16ba-4121-89d1-342f6e039a6d

fn main() {
    let s1 = String::from("hello");

    let len = calculate_length(&s1);

    println!("The length of '{}' is {}.", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
//    s.push_str(", world!") //报错
}
  • 函数形参 &String
  • 引用可以保持值的所有权不移动
  • 引用指向的值,默认不可改变
  • 多个不可变引用可以同时存在

可变引用

  • 加 mut 为 &mut
fn main() {
    let mut s = String::from("hello");
    change(&mut s);
}

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}

  • 可变引用同时只能存在一个

      let mut s = String::from("hello");
  let r1 = &mut s;
  let r2 = &mut s; //报错
  println!("{}, {}", r1, r2);

    防止数据竞争

  • 可变与不可变引用不能同时存在

      let mut s = String::from("hello");

  let r1 = &s; // 没问题
  let r2 = &s; // 没问题
  let r3 = &mut s; // 大问题

  println!("{}, {}, and {}", r1, r2, r3);

    不可变引用可不想自己的值被可变引用改变

    • 引用的生命周期从借用处开始,一直持续到最后一次使用的地方
      重点在于在可变引用发生后不能再使用不可变引用,即不可在 r3 定义后再使用 r1,r2,如果只使用 r3,是可以通过编译的,这个属于编译器优化

  • 引用作用域的位置是最后一次使用的位置

      fn main() {
     let mut s = String::from("hello");
 
     let r1 = &s;
     let r2 = &s;
     println!("{} and {}", r1, r2);
     // 新编译器中，r1,r2作用域在这里结束

     let r3 = &mut s;
      println!("{}", r3);
} // 老编译器中，r1、r2、r3作用域在这里结束
  // 新编译器中，r3作用域在这里结束
