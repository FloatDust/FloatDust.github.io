引入机制

WorkSpace(大型项目)

可以进一步将多个包联合在一起,组成工作空间

Package(项目)

  • 有一个或多个 crate
    • 最多一个库,任意多二进制.
      二进制文件通常位于 ./src/bin
  • 包含一个 Cargo.toml 文件
    • toml 文件中有[package]描述

Crate(包)

Crate 是一个单独的可编译单元,可编译成库或者可执行文件

二进制(可执行文件)

  • 以 ./src/main.rs 为原点(crate root)编译 crate
  • 可编译为可执行程序
    • 每个 ./src/bin 下文件都会被编译成独立的二进制文件

库(Library)

  • 以 ./src/lib.rs 为原点编译 crate
  • 大多数 crate 都是指库

  • 静态库与动态库

    • 区别

    静态库在编译时会被编译进程序内部
    动态库不会被编译进去

    • 优缺点

    静态库被打包后,加载速度快,可独立运行
    动态库方便更新

mod

默认情况下，所有的类型都是私有化的，包括函数、方法、结构体、枚举、常量，是的，就连模块本身也是私有化的

模块树

// 餐厅前厅，用于吃饭
mod front_of_house {
    mod hosting {
        fn add_to_waitlist() {}

        fn seat_at_table() {}
    }

    mod serving {
        fn take_order() {}

        fn serve_order() {}

        fn take_payment() {}
    }
}

mod.rs

  • mod.rs 放入不同的文件夹下面,起到管理文件夹下不同模块可见性的作用
  • 或者在 ./src/ 目录下放入与文件夹同名的 rs 文件

mod 关键字

相当于 import,加载其他文件的 mod

  • 只有在 main.rs 文件中，才需要 mod 引入其他模块
    在其他 rs 文件中，只需要用 use 引入其他模块函数

use

使用 use 关键字把路径提前引入到当前作用域中，随后的调用就可以省略该路径

mod front_of_house {
    pub mod hosting {
        pub fn add_to_waitlist() {}
    }
}

use front_of_house::hosting::add_to_waitlist;

pub fn eat_at_restaurant() {
    add_to_waitlist();
    add_to_waitlist();
    add_to_waitlist();
}

  • 模块再导出 pub use
    当外部的模块项 A 被引入到当前模块中时，它的可见性自动被设置为私有的，如果你希望允许其它外部代码引用我们的模块项 A，那么可以对它进行再导出

      pub use crate::front_of_house::hosting;

  • 简化引入 use crate::{}

      use std::collections::HashMap;
  use std::collections::BTreeMap;
  use std::collections::HashSet;

  use std::io;
  use std::io::Write;

    可以简化为

      use std::collections::{HashMap,BTreeMap,HashSet};
  use std::{self,Write};

  • 关键字 as

      use std::io::Result as IoResult

pub

  • pub ：公共的，任何人都可以访问。
  • pub(crate) ：在当前 crate 内是公共的，但对外部 crate 不可见。
  • pub(in path) ：在指定路径内的模块是公共的。
  • pub(self) ：在当前模块内是公共的。
  • pub(super) ：在当前模块的父模块内是公共的。

path

绝对路径

从包根开始,以包名或者 crate 作为开头.
crate::front_of_house::hosting::add_to_waitlist();

相对路径

从当前模块开始,以当前模块或者 self,super 作为开头
front_of_house::hosting::add_to_waitlist();
