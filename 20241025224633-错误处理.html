错误处理

  • 错误分类
  • 可恢复错误 - 使用 Result<T, E> 处理
  • 不可恢复错误 - 使用 panic! 处理

可恢复错误

enum Result<T, E> {
    Ok(T),
    Err(E),
}

use std::fs::File;

fn main() {
    let f = File::open("hello.txt");

    let f = match f {
        Ok(file) => file,
        Err(error) => {
            panic!("Problem opening the file: {:?}", error)
        },
    };
}

不可恢复错误

主线程 main 不适合处理太多逻辑,
支线程错误,只会停止支线程

fn main() {
    panic!("crash and burn");
}

unwrap

从 Option 或 Result
类型中获取值,成功则返回值，失败则 panic，总之不进行任何错误处理

use std::net::IpAddr;
let home: IpAddr = "127.0.0.1".parse().unwrap();
  • unwrap_or(default) 如果失败则返回一个默认值

expect

expect 跟 unwrap 很像，也是遇到错误直接 panic, 但是会带上自定义的错误提示信息

use std::fs::File;

fn main() {
    let f = File::open("hello.txt").expect("Failed to open hello.txt");
}

可以将 Some 转换为 String

let some_value: Option<String> = Some("Hello".to_string());
let result: String = some_value.expect("This should never be None");
  • 如果 some_value 是 Some(String)，expect 会直接返回 String 值。
  • 如果 some_value 是 None，expect 会恐慌，程序会停止执行，并打印出"This should never be None"
