特征

基础

pub trait Summary {
    fn summarize(&self) -> String;
}
pub struct Post{...}
impl Summary for Post {
    fn summarize(&self) -> String{...}
}
  • 特征只定义行为看起来是什么样的，而不定义行为具体是怎么样的
  • impl Summary for Post ，读作“为 Post 类型实现 Summary 特征”

孤儿规则

如果你想要为类型 A 实现特征 T，那么 A 或者 T 至少有一个是在当前作用域中定义的！

默认实现

pub trait Summary {
    fn summarize(&self) -> String {
        String::from("(Read more...)")
    }
}

特征约束

pub fn notify<T: Summary<T,Output = T>>(item: &T) -> T {...}
  • item 这个参数实现了 Summary 特征,
  • Output 指定返回结果也是 T 类型

优势

  1. pub fn notify(item1: &impl Summary, item2: &impl Summary) {}
  2. pub fn notify<T: Summary>(item1: &T, item2: &T) {}

特征约束可以使参数 item1 和 item2 都拥有相同的参数(结构体),而<1>中参数则可有不同的结构体传入

多重约束

pub fn notify(item: &(impl Summary + Display)) {}
或者使用特征约束的形式
pub fn notify<T: Summary + Display >(item: &T) {}

where 约束

fn some_function<T: Display + Clone, U: Clone + Debug>(t: &T, u: &U) -> i32 {}

where 可以简化特征约束

fn some_function<T, U>(t: &T, u: &U) -> i32
    where T: Display + Clone,
          U: Clone + Debug
{}

使用约束特征有条件的实现方法

struct Pair<T>{}
impl<T> Pair<T> {}
impl<T: Display + PartialOrd> Pair<T>{
    fn cmp_display(&self){}
}

是有实现了 Display 和 PartialOrd 特征的 Pair 结构体才能拥有 cmp_display 函数

有条件的实现特征

impl<T: Display> ToString for T {
    // --snip--
}

为实现 Display 特征的结构体,再实现 ToString 函数

返回特征

fn returns_summarizable() -> impl Summary{ some_struct_impl_summary{} }

返回一个实现了 Summary 特征的结构体

派生特征

被 derive 标记的对象会自动实现对应的默认特征代码，继承相应的功能

特征对象

定义

  • 特征对象指的是实现了某个特征的类型实例(结构体)
  • 只有在运行时,才能知道调用什么方法, 动态分发
pub trait Draw {
    fn draw(&self);
}
pub struct Button {...}
impl Draw for Button{
    fn draw(&self){}
}
pub struct Box{...}
impl Draw for Box{
    fn draw(&self){}
}

这里的 Button 和 Box 就是待实例化的特征对象

pub struct Screen {
    pub components: Vec<Box<dyn Draw>>,
}
impl Screen {
    pub fn run(&self) {
        for component in self.components.iter() {
            component.draw();
        }
    }
}

componets 就是特征对象的修饰,实现了特征 Draw 的动态数组
dyn 关键字动态特征对象
利用 run 方法来包裹 draw 函数

fn main() {
    let screen = Screen {
        components: vec![
            Box::new(SelectBox {...}),
            Box::new(Button {...}),
        ],
    };

    screen.run();
}

实例化 Screen 屏幕,利用动态数组,来分别执行每个

限制

  • 方法的返回类型不能是 Self
  • 方法没有任何泛型参数

例子:

pub struct Screen {
    pub components: Vec<Box<dyn Clone>>,
}

关联类型

  • 比泛型简单,增加可读性
pub trait Iterator {
    type Item;

    fn next(&mut self) -> Option<Self::Item>;
}
  • 由 type 定义
impl Iterator for Counter {
    type Item = u32;

    fn next(&mut self) -> Option<Self::Item> {
        // --snip--
    }
}

fn main() {
    let c = Counter{..}
    c.next()
}

默认泛型类型参数

trait Add<RHS=Self> {
    type Output;

    fn add(self, rhs: RHS) -> Self::Output;
}

不同特征同名方法

  • 初始化
trait Pilot{ fn fly(&self) }
trait Wizard{ fn fly(&self) }
Struct Human;
  • 使用
fn main() {
    let person = Human;
    Pilot::fly(&person); // 调用Pilot特征上的方法
    Wizard::fly(&person); // 调用Wizard特征上的方法
    person.fly(); // 调用Human类型自身的方法
}
  • 完全限定语法
    <Type as Trait>::function(receiver_if_method, next_arg, ...);
    如果特征方法参数没有 self 的时候(关联函数),就使用完全限定语法
trait Animal{ fn baby_name() }
Struct Dog;
impl Dog{ fn baby_name() }
impl Animal for Dog{ fn baby_name() }

fn main() {
    <Dog as Animal>::baby_name();
}
