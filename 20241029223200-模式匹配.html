变量解构

fn main() {
    //将三个元素组合到一起形成一个tuple
    let t = (1, 's', "白质".to_string());
    //刚好与以上相反，把一个组合数据结构，拆解成三个不同的变量
    let (t_one, t_two, t_tree) = t;
    println!("t_one: {}, t_two: {}, t_tree: {}", t_one, t_two, t_tree);
}

利用 let 解构

变量遮蔽

使用 let 修饰相同的变量名,后面声明的变量会遮蔽到前面的声明.

// 字符串类型
let spaces = "   ";
// usize数值类型
let spaces = spaces.len();
  • 涉及到内存再分配,没有 mut 修饰的变量性能好
  • 优势在于不用废弃之前的变量,使用旧名字,不用去想更多的名字

常量

const MAX_POINT: u32 = 100_000;

  • 常量不允许使用 mut,自始至终不可变
  • 常量使用 const 来声明

匹配

利用 match 来匹配,rust 的匹配是穷尽的,必须穷举所有的可能

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}
fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter => 25,
    }
}

if let 简洁控制流

  • 基本语法
if let pattern = expression {
    // 匹配成功时执行的代码块
} else {
    // 可选的 else 块，匹配失败时执行
}

匹配 Option

  let config_max = Some(3u8);
  match config_max {
      Some(max) => println!("The maximum is configured to be {}", max),
      _ => (),
  }

两个相等

  let config_max = Some(3u8);
  if let Some(max) = config_max {
      println!("The maximum is configured to be {}", max);
  }

匹配元组

  let triple = (0, -2, 3);
  // 试一试 ^ 将不同的值赋给 `triple`
  println!("Tell me about {:?}", triple);
  // match 可以解构一个元组
  match triple {
      // 解构出第二个和第三个元素
      (0, y, z) => println!("First is `0`, `y` is {:?}, and `z` is {:?}", y, z),
      (1, ..) => println!("First is `1` and the rest doesn't matter"),
      // `..` 可用来忽略元组的其余部分
      _ => println!("It doesn't matter what they are"),
      // `_` 表示不将值绑定到变量
  }

ref

创建引用

let ref _is_a_reference = 3;

这行将得到一个引用。

在 match 语句中使用 ref

let s = String::from("Hello");

match s {
    ref r => println!("r: {}", r),
}

// s 仍然可以使用，因为我们只是引用了它
println!("s: {}", s);

在这个例子中，ref r 匹配 s，但 r 是一个引用，指向 s，而不是 s 本身。
