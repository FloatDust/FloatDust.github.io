函数式编程

闭包

基础

param1, param2,… {

语句 1;
语句 2;
返回表达式
}

  • 闭包的多种形式
fn  add_one_v1   (x: u32) -> u32 { x + 1 }
let add_one_v2 = |x: u32| -> u32 { x + 1 };
let add_one_v3 = |x|             { x + 1 };
let add_one_v4 = |x|               x + 1  ;

闭包是一种匿名函数，它可以赋值给变量也可以作为参数传递给其它函数，不同于函数的是，它允许捕获调用者作用域中的值

fn main() {
   let x = 1;
   let sum = |y| x + y;

    assert_eq!(3, sum(2));
}

y 是入参, x 是作用域中的参数

  • 简化代码
  • 针对没有使用的闭包,编译器会提示你为闭包参数添加类型标注,因为它缺乏必要的上下文.
  • 当编译器推导出一种类型后，它就会一直使用该类型
  • 闭包自动实现 Copy 特征的规则是，只要闭包捕获的类型都实现了 Copy 特征的话，这个闭包就会默认实现 Copy 特征。

结构体中的闭包

struct Cacher<T>
where
    T: Fn(u32) -> u32,
{
    query: T,
    value: Option<u32>,
}

Fn(u32) -> u32 该闭包拥有一个 u32 类型的参数，同时返回一个 u32 类型的值。Fn 为闭包特征

闭包作为函数返回值

fn factory(x:i32) -> Box<dyn Fn(i32) -> i32> {
    let num = 5;

    if x > 1{
        Box::new(move |x| x + num)
    } else {
        Box::new(move |x| x - num)
    }
}

Box<dyn Fn(i32) -> i32> 代表 |x| x + num

闭包作为函数参数

  • 闭包的类型
    闭包实现了哪种 Fn 特征取决于该闭包如何使用被捕获的变量
  • 一个闭包不仅仅实现某种 Fn 特征,规则如下:
    • 所有的闭包都自动实现了 FnOnce 特征，因此任何一个闭包都至少可以被调用一次
    • 没有移出所捕获变量的所有权的闭包自动实现了 FnMut 特征
    • 不需要对捕获变量进行改变的闭包自动实现了 Fn 特征

FnOnce 转移所有权(移动)

fn fn_once<F>(func: F)
where
    F: FnOnce(usize) -> bool,
{
    println!("{}", func(3));
    println!("{}", func(4));//出错
}

fn main() {
    let x = vec![1, 2, 3];
    fn_once(|z|{z == x.len()})
}

FnMut 可变引用

可以多次调用

fn main() {
    let mut s = String::new();

    let update_string =  |str| s.push_str(str);

    exec(update_string);

    println!("{:?}",s);
}

fn exec<'a, F: FnMut(&'a str)>(mut f: F)  {
    f("hello")
}

不需要加 mut let update_string 就可以通过编译

Fn 

fn main() {
    let s = "hello, ".to_string();

    let update_string =  |str| println!("{},{}",s,str);

    exec(update_string);

    println!("{:?}",s);
}

fn exec<'a, F: Fn(String) -> ()>(f: F)  {
    f("world".to_string())
}

s 为不可变引用

move 关键字

fn main() {
    let s = String::new();

    let update_string =  move || println!("{}",s);

    exec(update_string);
}

fn exec<F: Fn()>(f: F)  {
    f()
}
