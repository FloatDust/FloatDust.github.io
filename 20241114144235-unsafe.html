Unsafe

基础

fn main() {
    let mut num = 5;
    let r1 = &num as *const i32;

    unsafe {
        println!("r1 is: {}", *r1);
    }
}
  • 越小的 unsafe 边界越好

使用 unsafe 的原因

  1. 通过编译器检查
  2. 访问底层硬件,进行系统编程

Unsafe 能力

  1. 解引用裸指针
  2. 调用 unsafe 或外部函数
  3. 访问修改一个可变的静态变量
  4. 实现一个 unsafe 特征
  5. 访问 union 中的字段

裸指针

可变裸指针 *mut T ,不可变裸指针 *const T

let mut num = 5;

let r1 = &num as *const i32;//不可变裸指针
let r2 = &mut num as *mut i32;//可变裸指针
  • 可以绕过 Rust 的借用规则，可以同时拥有一个数据的可变、不可变指针，甚至还能拥有多个可变的指针
  • 并不能保证指向合法的内存
  • 可以是 null
  • 没有实现任何自动的回收 (drop)

调用 unsafe 函数或方法

unsafe fn dangerous() {}
fn main() {
    unsafe{
        dangerous();
    }
}

访问修改可变的静态变量

static mut REQUEST_RECV: usize = 0;
fn main() {
   unsafe {
        REQUEST_RECV += 1;
        assert_eq!(REQUEST_RECV, 1);
   }
}

使用 unsafe 语句块才能访问和修改 static 变量

实现 unsafe 特征

unsafe trait Foo {
    // 方法列表
}

unsafe impl Foo for i32 {
    // 实现相应的方法
}

fn main() {}

之所以会有 unsafe 的特征，是因为该特征至少有一个方法包含有编译器无法验证的内容

访问 union 中的字段

#[repr(C)]
union MyUnion {
    f1: u32,
    f2: f32,
}

union 主要用于与 c 代码进行交互

FFI

调用其他语言函数

extern "C" {
    fn abs(input: i32) -> i32;
}

fn main() {
    unsafe {
        println!("Absolute value of -3 according to C: {}", abs(-3));
    }
}

其中 C 是应用二进制接口 ABI.在所有的 ABI 中,C 语言是最常见的.

使 rust 被其他语言调用

#[no_mangle]
pub extern "C" fn call_from_c() {
    println!("Just called a Rust function from C!");
}

内联汇编

可以在 rust 代码中嵌入汇编代码

use std::arch::asm;

let x: u64;
unsafe {
    asm!("mov {}, 5", out(reg) x);
}
assert_eq!(x, 5);
